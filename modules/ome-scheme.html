Scheme

Scheme is my favourite programming language, it is small, elegant and powerful. Unfortunately, there're too many choices for scheme implementation, which makes it a headache when you want to do some real world job with scheme.

Oh-my-emacs adopts geiser, which provides a similar SLIME way to work with scheme. To install geiser, you need install automake and autoconf. And to run geiser, you need install at least one of the geiser supported scheme implementation, namely, guile or racket.

There's a new experimental ac-geiser project, written by me — yeah, it's me, I am the author, which provides a auto-complete backend for geiser. It is still in beta version, however, and will be sent to el-get when stable. 

(defun ome-geiser-setup ()
  (setq geiser-default-implementation 'guile))

(defun ome-ac-geiser-setup ()
  (add-hook 'geiser-mode-hook 'ac-geiser-setup)
  (add-hook 'geiser-repl-mode-hook 'ac-geiser-setup)
  (eval-after-load "auto-complete"
    '(add-to-list 'ac-modes 'geiser-repl-mode)))

(when (or (executable-find "guile") (executable-find "racket"))
  (ome-install 'geiser)
  (ome-install 'ac-geiser))

;; (ome-install 'quack)

Actually, the official SLIME contrib has some swank backends for scheme implementations, you can try it if you like. There's a swank-chicken for chicken scheme, but it only provides a small part of SLIME features. So it is disabled by default. 

(defun ome-swank-chicken-setup ()
  (setq swank-chicken-path "~/.emacs.d/el-get/swank-chicken/swank-chicken.scm")
  (add-hook 'scheme-mode-hook
            (lambda ()
              (slime-mode t))))

(ome-install 'swank-chicken)
